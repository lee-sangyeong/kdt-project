# 대구광역시 창업 컨설턴트

대구광역시의 인구, 점포 수, 공시지가 데이터를 결합해 업종별 창업 블루오션 지역을 찾는 SQL 미니 프로젝트입니다.  
단순히 유동인구가 많은 지역을 추천하는 방식이 아니라, **인구 대비 점포 수**, **법정동별 공시지가**, **점포 밀도**를 함께 비교해 실제 창업 관점에서 진입 가능한 후보지를 도출하는 것을 목표로 했습니다.

> 이 README는 팀 프로젝트 전체 흐름을 설명하되, **이상영이 담당한 고기집 분석 파트**를 중심으로 정리했습니다.

---

- **프로젝트명**: 대구광역시 창업 컨설턴트
- **프로젝트 유형**: SQL 미니 프로젝트
- **진행 기간**: 2026.02.10 ~ 2026.02.11 (1일)
- **팀원**: 정유진, 장윤진, 이상영, 이재헌

이 프로젝트는 “유명한 상권”이 아니라 **실제 수익성이 높을 가능성이 있는 대구 지역의 창업 후보지**를 찾기 위해 진행했습니다.  
업종별 점포 데이터와 행정구역별 인구, 공시지가를 함께 활용하여, 공급 과잉 지역은 피하고 경쟁 강도와 임대 부담이 상대적으로 낮은 지역을 찾는 방식으로 분석을 설계했습니다.

## 2. 프로젝트 배경 / 문제 정의

창업 입지 판단에서 단순 유동인구나 유명 상권만 기준으로 삼으면, 이미 경쟁이 과도하게 몰린 지역을 선택할 가능성이 높습니다.  
따라서 본 프로젝트에서는 다음 질문에 답하고자 했습니다.

1. 대구 내에서 업종별로 **인구 대비 점포 수가 낮아 공급 여유가 있는 지역**은 어디인가?
2. 같은 업종이라도 지역별로 **경쟁 강도와 임대료 부담**은 어떻게 다른가?
3. 점포 수만 적은 것이 아니라 **공시지가까지 함께 고려했을 때 실제 진입 가능한 후보지**는 어디인가?

## 5. 분석 기준 / 핵심 지표

프로젝트 전체의 핵심 분석 기준은 아래 세 가지였습니다.

### 5-1. 인구수 당 점포수
해당 업종의 구·동 단위별 인구 대비 점포 수를 계산해 공급 여유를 확인했습니다.

### 5-2. 법정동별 공시지가
동 단위 후보지를 좁힐 때 공시지가를 함께 비교해 임대료 부담 가능성을 반영했습니다.

### 5-3. 점포수 당 구 면적
구 단위에서 점포 밀도를 확인해 특정 업종이 어느 정도 집중되어 있는지 비교했습니다.

### 5-4. 이상영 파트의 추가 해석 지표
고기집 분석에서는 아래 지표를 중심으로 해석했습니다.

- `people_per_shop`: 점포 1개가 상대해야 하는 잠재 고객 규모
- `shops_per_10k`: 인구 1만 명당 고기집 수
- 동별 평균 공시지가
- 동별 고기집 수
- 공시지가 × 점포 수 기준 4분면 분류

4. 업종별로 구·군 단위 경쟁 강도 분석
5. 후보 지역의 동 단위 공시지가 비교
6. 최종 후보 지역 도출 및 발표

## 6-2. 이상영 담당 분석 흐름: 고기집 창업 분석

제가 맡은 파트는 **대구 내 고기집 창업 후보지 도출**이었습니다.  
분석 흐름은 다음 순서로 정리했습니다.

### 1) 대구 전체 고깃집 현황 확인
먼저 대구 전체 고깃집 분포를 확인해 어느 구·군에 점포가 집중되어 있는지 파악했습니다.

### 2) 구·군별 경쟁 강도 분석
`people_per_shop` 지표를 통해 “점포 하나가 상대해야 하는 잠재 고객 수”를 비교했습니다.  
이 지표를 통해 단순 점포 수가 아니라 **상대적 경쟁 강도**를 해석했습니다.

### 3) 구·군별 점포 밀도 분석
`shops_per_10k` 지표를 사용해 인구 1만 명당 고기집 수를 비교했습니다.  
같은 점포 수라도 인구 규모에 따라 포화 수준이 다를 수 있기 때문에 보완 지표로 사용했습니다.

### 4) 동별 공급 부족 지역 확인
구·군 단위에서 후보지를 좁힌 뒤, 동 단위로 내려가 고깃집 공급이 부족한 지역을 찾았습니다.  
공급이 거의 없는 동은 주민 소비가 외부 상권으로 유출될 가능성이 있다고 해석했습니다.

### 5) 공시지가 확인
동별 평균 공시지가를 함께 비교해, 점포 수가 적더라도 임대료 부담이 과도한 지역은 제외하는 방향으로 검토했습니다.

### 6) 4분면 시각화
최종 결과 CSV를 Python으로 불러와 아래 두 기준으로 4분면 분석을 수행했습니다.

- x축: 동별 평균 공시지가
- y축: 동별 고기집 수

여기에 동별 인구를 점 크기로 반영해, **입지 비용 / 경쟁 강도 / 잠재 수요**를 동시에 해석할 수 있도록 시각화했습니다.

## 7. SQL 설계 / 쿼리 관점에서의 정리

현재 업로드 자료 기준으로는 **최종 SQL 원문 전체가 직접 제공되지는 않았습니다.**  
다만 발표자료, 결과 CSV, Python 후처리 코드를 통해 아래 흐름은 확인할 수 있었습니다.

- `meat_shop`을 기준으로 고기집 점포 데이터를 관리
- 구·군 단위 인구 테이블과 연결하여 경쟁 강도 집계
- 동 단위 인구 / 공시지가 데이터와 결합
- 동별 `shop_count`, `population`, `avg_land_price`를 산출
- 최종 결과를 CSV로 저장한 뒤 Python에서 후처리 및 시각화 수행

즉, **SQL에서는 지역 단위 집계와 결합**, **Python에서는 해석용 지표 추가와 시각화**를 담당하는 구조로 정리할 수 있습니다.

---

## 8. Python 후처리 / 시각화

업로드된 Python 파일 기준으로, 최종 결과 CSV를 불러온 뒤 다음 작업을 수행했습니다.

1. `avg_land_price`와 `shop_count`의 전체 평균 계산
2. 평균 공시지가와 평균 점포 수를 기준으로 동별 4분면 분류
3. `people_per_shop = population / shop_count` 지표 추가
4. 4분면 기준으로 결과 정렬 및 CSV 재저장
5. 구·군별 색상 구분, 인구 규모를 점 크기로 둔 산점도 시각화
6. 공시지가 축을 로그 스케일로 두어 지역 간 가격 차이를 비교 가능하게 조정

이를 통해 발표용 결론이 아니라, **실제 후보지를 비교 설명할 수 있는 시각적 근거**를 만들었습니다.

## 10. 주요 결과

### 10-1. 팀 프로젝트 전체 결과
업종별로 단순한 중심 상권이 아니라, **공급 과잉 여부와 입지 비용을 함께 고려한 후보지**를 도출했습니다.

### 10-2. 이상영 파트 주요 결과
고기집 분석에서는 다음 결과를 도출했습니다.

- 대구 전체 고깃집 수는 **북구 1위, 동구 2위, 달서구 3위**
- `people_per_shop` 기준으로 **수성구는 상대적으로 경쟁 여유가 있고, 북구는 경쟁이 치열**
- `shops_per_10k` 기준으로 **북구는 높고 수성구는 낮아**, 수성구가 상대적으로 덜 포화된 지역으로 해석 가능
- 동 단위로 보면 공급 부족 지역이 존재하며, 이는 주민 소비가 외부 상권으로 유출될 가능성을 시사
- **동성로는 공시지가가 매우 높아 임대료 부담이 큰 지역**으로 판단
- 최종적으로 **동구와 수성구를 우선 추천 지역**으로 도출

### 10-3. 세부 후보 지역
- **동구 도심권**: 율암동, 봉무동, 서호동
- **동구 외곽권**: 매여동, 미곡동, 미대동
- **수성구**: 가천동, 고모동, 매호동

추가로 4분면 분석 기준에서는 **저가·저밀 구간의 최적 후보 동 개수**가 동구 30개, 수성구 10개로 확인되어 동구가 더 넓은 후보군을 보였고, 수성구는 상대적으로 선별된 후보지가 나타났습니다.

## 11. 인사이트 및 결론

이 프로젝트에서 확인한 핵심은 **점포 수가 적다고 해서 바로 창업 적지가 되는 것은 아니라는 점**입니다.  
고기집은 경쟁 업종이기 때문에 아래 세 조건을 함께 봐야 했습니다.

1. 인구 대비 점포 수가 과도하지 않은가
2. 동 단위에서 공급 공백이 존재하는가
3. 공시지가가 지나치게 높지 않은가

이 기준으로 보면, 중구 중심 상권처럼 이미 상업성이 높은 지역은 진입 장벽도 높은 반면, 동구와 수성구 일부 지역은 **비용 부담과 경쟁 강도를 함께 고려했을 때 현실적인 후보지**로 해석할 수 있었습니다.

즉, 본 프로젝트는 “핫플레이스 추천”이 아니라, **데이터 기준으로 경쟁 과열을 피하고 상대적으로 진입 가능한 지역을 찾는 창업 분석**에 가깝습니다.

## 12. 아쉬운 점 / 개선 방향

### 한계
- 상권 분석 기준이 인구, 공시지가, 점포 수에 집중되어 있어 유동인구·매출·연령대 분포까지 반영하지는 못했습니다.
- 현재 자료 기준으로 SQL 원문 전체가 정리되어 있지 않아, 쿼리 수준의 재현성은 보완이 필요합니다.
- 공시지가는 임대료의 대리 지표로 활용했지만, 실제 상가 임대료와는 차이가 있을 수 있습니다.
- 점포 수가 적은 지역은 공급 부족일 수도 있지만, 수요 자체가 낮은 지역일 가능성도 함께 검토해야 합니다.

### 개선 방향
- 카드 매출, 유동인구, 연령대 소비 데이터 추가
- 실제 상가 임대료 데이터와 결합
- SQL 스크립트 별도 정리로 재현성 강화
- Streamlit 또는 대시보드 형태로 시각화 확장
- 업종별 비교 기준을 통일해 추천 로직을 일반화
